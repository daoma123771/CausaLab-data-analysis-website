# CausaLab V1.0 完整使用说明

## 1. 软件简介

CausaLab V1.0 是一套面向 CSV/Excel 表格数据的统计分析向导平台。用户上传房价、工资、油耗、问卷或实验记录等二维表格后，系统会自动完成数据画像、字段类型识别、统计方法推荐、样本量与数据条件判断，并进一步输出统计检验、回归建模、可视化图表、结果表格和可复核分析报告。

本系统适合用于统计学课程实验、数据分析练习、A/B 实验结果评估以及小型数学建模任务。它的重点不是要求用户一开始就知道应该选择哪一种统计方法，而是先读取数据结构，再引导用户确认分析目标和变量选择，最后生成可解释的统计结果。

## 2. 运行环境

推荐环境如下：

- 操作系统：Windows 10/11
- Python：3.12.4
- Node.js：24.18.0
- npm：11.16.0
- 浏览器：Chrome、Edge 或 Codex 内置浏览器

项目主要依赖：

- 后端：FastAPI、Pandas、NumPy、SciPy、statsmodels、python-docx
- 前端：Vue 3、TypeScript、Vite、ECharts

## 3. 项目目录说明

```text
CausaLab/
├─ backend/                  后端服务、接口和统计计算逻辑
│  ├─ app/api/               FastAPI 接口定义
│  ├─ app/services/          样本量、数据诊断、统计分析、报告生成服务
│  ├─ app/schemas/           请求和响应数据结构
│  └─ tests/                 后端自动化测试
├─ frontend/                 前端 Vue 项目
│  ├─ src/components/        工作台、分析向导、报告等页面组件
│  ├─ src/App.vue            页面导航与整体布局
│  └─ src/style.css          页面样式
├─ docs/                     项目说明、算法说明、测试用例和软著材料
├─ examples/                 示例数据和示例报告
├─ scripts/                  辅助脚本
├─ output/                   本地导出材料，默认不上传 Git
└─ README.md                 项目总说明
```

其中 `backend/` 和 `frontend/` 是程序核心代码；`docs/` 是课程报告、软著材料和说明文档的重要来源；`examples/` 中的示例数据可用于课堂演示。

## 4. 首次安装依赖

如果老师或其他使用者第一次拿到压缩包，需要先进入项目目录安装依赖。以下命令中的路径可以替换成实际解压位置。

### 4.1 安装后端依赖

```bat
cd /d F:\2026小学期项目
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```

如果项目压缩包中已经带有可用虚拟环境，可跳过创建虚拟环境步骤，直接使用 `.venv\Scripts\python.exe` 启动后端。

### 4.2 安装前端依赖

```bat
cd /d F:\2026小学期项目\frontend
npm install
```

如果本机不能直接识别 `npm`，但 Node.js 安装在 `E:\`，可以使用：

```bat
E:\npm.cmd install
```

## 5. 启动系统

系统需要同时启动后端和前端，因此建议打开两个命令行窗口。

### 5.1 启动后端

第一个 CMD 窗口输入：

```bat
cd /d F:\2026小学期项目\backend
F:\2026小学期项目\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

看到类似下面内容表示后端启动成功：

```text
Uvicorn running on http://127.0.0.1:8000
```

注意：这个窗口不要关闭。关闭后，前端会无法请求统计分析接口。

### 5.2 启动前端

第二个 CMD 窗口输入：

```bat
cd /d F:\2026小学期项目\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

如果本机 npm 路径为 `E:\npm.cmd`，可以输入：

```bat
cd /d F:\2026小学期项目\frontend
E:\npm.cmd run dev -- --host 127.0.0.1 --port 5173
```

看到下面内容表示前端启动成功：

```text
Local: http://127.0.0.1:5173/
```

### 5.3 打开网页

在浏览器地址栏输入：

```text
http://127.0.0.1:5173/
```

页面打开后，即可进入 CausaLab 工作台。

## 6. 功能使用说明

### 6.1 工作台

工作台用于展示系统主流程和功能入口。主流程为：

```text
导入数据 → 数据画像 → 方法推荐 → 样本量判断 → 统计检验/建模 → 结果归档
```

建议首次演示时从“分析向导”进入，因为该入口最能体现系统的连续分析流程。

### 6.2 分析向导

分析向导是系统的核心页面。使用步骤如下：

1. 点击左侧“分析向导”。
2. 点击“选择数据文件”。
3. 上传 CSV 或 XLSX 表格，例如 `examples/housing_sample.csv`。
4. 系统自动识别数据行列数、字段类型、目标变量候选、分组变量候选和数据质量评分。
5. 系统推荐可开展的统计分析路线，包括描述统计、相关分析、均值比较、比例检验和线性回归。
6. 用户选择一个分析任务，确认目标变量、分组字段或解释变量。
7. 点击“运行统计分析 / 建模”。
8. 查看统计指标、可视化图表、结果表格和文字解释。

示例数据 `housing_sample.csv` 包含以下字段：

| 字段 | 含义 |
|---|---|
| district_id | 区域编号 |
| median_house_price | 房价中位数 |
| median_income | 收入中位数 |
| avg_rooms | 平均房间数 |
| house_age | 房龄 |
| distance_to_center | 距中心距离 |
| region | 区域类别 |

推荐演示任务：

- 选择“建立连续目标变量的解释模型”，以 `median_house_price` 为目标变量，选择收入、房间数、房龄、距离等变量作为解释变量，运行线性回归。
- 选择“比较多组均值是否存在差异”，以 `region` 或 `avg_rooms` 为分组字段，以 `median_house_price` 为数值指标，运行 ANOVA。
- 选择“分析两个数值变量之间是否相关”，查看两个连续变量之间的 Pearson 和 Spearman 相关关系。

### 6.3 样本量规划

样本量规划用于实验前估算需要多少样本。用户可以选择比例指标或连续指标：

- 比例指标：适用于转化率、点击率、通过率等二元结果。
- 连续指标：适用于收入、成绩、时长、价格等连续结果。

输入基准水平、最小可检测效应、显著性水平、统计功效和分组比例后，系统会输出建议样本量、预计实验周期和功效曲线。

### 6.4 数据诊断

数据诊断用于检查上传数据是否适合进一步分析。系统会识别：

- 缺失值比例
- 完全重复行
- IQR 异常值
- 分组字段
- 分组比例失衡
- 综合质量评分

可使用 `examples/ab_test_sample.csv` 演示该功能，因为该文件包含缺失、重复和异常值，适合展示诊断效果。

### 6.5 A/B 效应评估

A/B 效应评估用于实验后判断两组差异是否显著。系统支持：

- 比例指标：两比例 Z 检验
- 连续指标：Welch t 检验
- Bootstrap 置信区间
- 效应量、p 值、统计决策和文字解释

该页面适合展示统计学假设检验、置信区间和稳健性验证的关系。

### 6.6 分析报告

分析报告页面用于将统计结果整理为 Word 文档。用户填写项目名称、分析人员、核心指标、统计方法、样本量、p 值、置信区间、统计决策、结论和风险备注后，可以点击“生成并下载 Word 报告”。

报告内容包括：

- 项目基础信息
- 核心统计结果
- 统计方法说明
- 数据质量评分
- 结论与风险提示

## 7. 课程演示建议

建议演示路径如下：

```text
工作台 → 分析向导 → 上传 housing_sample.csv → 查看数据画像 → 选择回归建模 → 运行分析 → 展示图表和结果表 → 分析报告
```

讲解时可以强调：

1. 用户只需要上传表格，系统会先识别数据结构。
2. 系统会根据字段类型推荐统计方法，而不是要求用户直接选择公式。
3. 分析结果包含统计量、p 值、图表、结果表格和文字解释。
4. 报告导出功能让分析过程可复核，适合作业提交和后续材料归档。

## 8. 常见问题

### 8.1 CMD 中输入 cd 后仍然在 C 盘

需要使用 `/d` 参数切换盘符：

```bat
cd /d F:\2026小学期项目\frontend
```

### 8.2 后端提示 No module named app

说明启动目录不对。后端必须在 `backend` 文件夹中启动：

```bat
cd /d F:\2026小学期项目\backend
F:\2026小学期项目\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

### 8.3 浏览器打不开 127.0.0.1:5173

一般是前端没有启动，或启动后窗口被关闭。请重新执行：

```bat
cd /d F:\2026小学期项目\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

### 8.4 页面能打开但分析失败

一般是后端没有启动。请检查 `http://127.0.0.1:8000/api/health` 是否可以返回系统状态。

### 8.5 上传 Excel 或 CSV 后没有推荐任务

可能原因包括：

- 表格为空或列名不规范；
- 数值字段太少；
- 分组字段唯一值过多；
- 缺失值过多导致有效样本不足。

建议先使用系统自带的 `examples/housing_sample.csv` 验证流程是否正常。
