# CausaLab V1.0 用户手册

## 1. 软件简介

CausaLab 智能实验设计与效应评估平台 V1.0 用于完成统计实验中的实验规划、数据质量诊断、A/B 效应评估、Bootstrap 稳健性验证和报告导出。用户无需手写统计代码，即可通过表单输入和上传数据获得可解释的实验结论。

## 2. 推荐使用流程

1. 进入“工作台”，查看系统能力和课程要求覆盖情况。
2. 进入“实验规划”，选择指标类型，填写基准水平、最小可检测效应、显著性水平和统计功效，生成建议样本量。
3. 进入“数据诊断”，上传 CSV 或 Excel 文件，查看字段角色、缺失、重复、异常值和分组比例诊断。
4. 进入“效应评估”，选择比例指标或连续指标，输入对照组与实验组数据，计算 p 值、效应量和置信区间。
5. 进入“分析报告”，填写项目名称、指标、样本量、结果和结论，导出 Word 报告。

## 3. 功能说明

### 3.1 工作台

工作台用于说明系统定位、推荐演示路径、核心模块和课程要求映射。该页面适合在课程答辩或软著介绍中作为系统入口展示。

### 3.2 实验规划

输入项包括：

- 指标类型：比例指标或连续指标；
- 基准水平：当前转化率、均值或标准差等；
- 最小可检测效应：希望实验能够识别的最小差异；
- 显著性水平：常用 0.05；
- 统计功效：常用 0.8 或 0.9；
- 分流比例：对照组和实验组流量比例。

输出项包括：

- 建议样本量；
- 预计总样本量；
- 可视化功效曲线；
- 参数解释与风险提示。

### 3.3 数据诊断

输入项：

- CSV 或 Excel 数据文件。

输出项：

- 数据行列数；
- 字段类型和角色推断；
- 缺失值数量；
- 重复行数量；
- 数值型异常值；
- 分组分布和失衡提示；
- 综合质量评分与诊断建议。

### 3.4 效应评估

比例指标支持输入对照组/实验组样本量和成功次数，系统执行两比例 z 检验并输出效应大小、p 值和置信区间。

连续指标支持输入原始观测值，系统执行 Welch t 检验并使用 Bootstrap 生成效应分布和置信区间。

### 3.5 分析报告

用户填写项目名称、分析人员、核心指标、统计方法和结论后，系统生成 Word 报告。报告内容包括：

- 报告元数据；
- 执行摘要；
- 核心统计结果；
- 置信区间；
- 方法与判断规则；
- 结论与使用建议。

## 4. 常见问题

### 上传文件失败怎么办？

请确认文件为 CSV、XLSX 或 XLS 格式，并检查文件是否为空、列名是否重复、数据是否被其他程序占用。

### p 值显著是否代表一定可以上线？

不是。统计显著只说明样本证据支持差异存在，上线决策仍需结合效应大小、实施成本、长期风险和数据质量。

### Bootstrap 的作用是什么？

Bootstrap 通过重复抽样观察效应估计的波动范围，用于辅助判断结果是否稳健，尤其适合解释连续指标和非正态数据场景。
